# 从零开始，用 Astro 搭建我的 GitHub Pages 个人网站

今天花了点时间，把我的个人网站从 0 到 1 搭起来了。在这里记录一下整个过程，也给想搭网站的朋友做个参考。

## 为什么要搭个人网站？

其实一直想有个自己的小站，可以放放自己做的小游戏、写的技术笔记、还有正在构思的小说。之前也用过一些博客平台，但总觉得不够自由，想加点什么功能都受限制。

干脆自己搭一个吧！

## 技术选型

选框架的时候纠结了一下，主要考虑这几点：

1. **静态站点** - 因为要部署在 GitHub Pages 上，静态站点是最合适的
2. **扩展性强** - 后面想做网页小游戏，最好能方便地集成 React/Vue 组件
3. **简单易上手** - 不想花太多时间在配置上

最后选了 **Astro**，原因很简单：
- 默认输出纯静态 HTML，速度快
- 支持 React/Vue/Svelte 等组件，后面做小游戏方便
- 配置简单，文件路由用起来很舒服
- 官方有 GitHub Pages 部署的教程

## 搭建过程

### 1. 初始化项目

用 Astro 的官方脚手架初始化项目，一行命令搞定：

```bash
npm create astro@latest
```

然后按照提示选配置，我选了比较基础的配置，后面慢慢加东西。

### 2. 规划网站结构

我的网站计划有这几个板块：

- 🎮 **网页小游戏** - 放我做的各种小游戏（第一个已经做好了：扫雷！）
- 📝 **博客** - 技术笔记、生活感悟、折腾记录
- 📚 **文档** - 一些项目文档和教程
- 📖 **小说** - 正在写的小说，慢慢更新

所以页面结构就是：首页 + 四个列表页。

### 3. 写页面

Astro 的文件路由真的很方便，`src/pages/` 目录下的文件自动对应路由。

比如：
- `src/pages/index.astro` → `/`
- `src/pages/games/index.astro` → `/games/`
- `src/pages/blog/index.astro` → `/blog/`

每个页面就是一个 `.astro` 文件，语法和 HTML 很像，还能在前面的 `---` 里写 JavaScript 代码，很灵活。

### 4. 公共组件

把导航栏、页脚这些公共部分抽成组件，放在 `src/components/` 和 `src/layouts/` 目录里。

比如 `Layout.astro` 就是公共布局，每个页面都用它，这样改导航栏的时候只改一个地方就行。

### 5. 样式

全局样式写在 `src/styles/global.css` 里，用 CSS 变量实现了深色模式自动跟随系统，不用额外写 JS。

配色选了比较清爽的蓝色系，浅色模式白色背景，深色模式深蓝灰背景，看着不累眼。

### 6. 部署到 GitHub Pages

这部分踩了个小坑 😅

最开始生成 GitHub Token 的时候只勾了 `repo` 权限，结果推送代码的时候报错，说 workflow 文件需要 `workflow` 权限。

重新生成 Token，多勾一个 `workflow` 权限就好了。

部署用的是 GitHub Actions，配置文件放在 `.github/workflows/deploy.yml`。每次 push 代码到 main 分支，就会自动构建并部署，大概 30 秒到 1 分钟就能更新好，很方便。

## 遇到的坑

### 1. Token 权限问题

刚才说了，推送 workflow 文件需要 `workflow` 权限，只勾 `repo` 是不够的。第一次弄的朋友注意一下。

### 2. 路径问题

GitHub Pages 部署的时候，如果是项目站点（不是用户名站点），路径会有个子路径，需要配置 `site` 和 `base`。

不过我用的是用户名站点（`gantang07.github.io`），就不用考虑这个问题了，省了不少事。
